# 系辦專案

### 專案總覽

- **系統定位：** 專為系辦設計的「活動報名」與「歷年紀錄」整合系統。
- **主要受眾：** 校內師生、校外一般大眾、系辦管理人員。
- **技術框架：** **Django (Python Web Framework)**，主打快速開發、高度安全性，並直接利用其強大的內建後台（Django Admin）降低開發成本。

### 一、 會員與身分模組 (Accounts)

系統具備帳號機制，以便長期追蹤使用者的參與履歷。

- **校內身分：** 具備專屬欄位（如學號、教職員編號），可透過學校信箱或 SSO 單一入口登入。
- **校外身分：** 透過一般 Email 註冊與驗證機制建立帳號。
- **參加履歷：** 系統自動綁定帳號與報名紀錄，使用者登入後可查看個人報名歷史。

### 二、 活動報名模組 (Events)

專注於流暢、精準的報名與報到流程，前期排除複雜的候補與硬體整合。

- **報名邏輯：** 採「先搶先贏」機制，額滿即自動關閉，**不設立候補機制**。
- **防超賣控管：** 採用資料庫層級的鎖定（或 Django 預設的 F() 表達式），確保在多人同時搶票時不會發生名額超發的問題。
- **現場報到：** 採「後台人工勾選」。活動當天，工作人員直接在系統後台對著名單打勾，完成出席紀錄。
- 報名能夠匿名，要保障資安(名字中間多一個圈)

### 三、 歷年紀錄與展示模組 (Records)

將過往的活動轉化為系所的數位資產與宣傳工具。

- **可見度設定：** 歷年紀錄牆「完全公開」，任何人（包含未登入的訪客）皆可瀏覽系上的活動成果與照片。
- **學制查詢設計：** 系統捨棄一般西元年，改採「學年度」**與**「學期」作為分類與前台查詢篩選的標準（例如：113學年度 上學期）。
- **多媒體建檔：** 每筆紀錄皆可上傳「活動成果摘要」、「活動首圖/照片」與「附件檔案」。

### 四、 後台維護與防呆機制 (Admin Features)

專為「非技術背景」的系辦助理與行政人員設計的友善管理環境。

- **基礎管理：** 直接深度客製化 Django Admin，提供直覺的中文介面來上架活動與管理名單。
- **上傳防呆與提示：** 針對照片與檔案實行嚴格的「大小限制」（如 5MB）。若助理上傳過大檔案，系統會直接在欄位旁顯示清晰的中文錯誤提示，而非伺服器錯誤代碼。
- **防誤刪（資源回收桶）：** 實作「軟刪除 (Soft Delete)」機制。後台按下的刪除僅是隱藏資料，工程師可隨時從資料庫完整救援，避免珍貴歷史紀錄遺失。
- **操作日誌 (Audit Log)：** 系統自動記錄所有後台行為（例：誰、在什麼時候、修改了哪一場活動的名額），方便系辦追蹤人為錯誤。
